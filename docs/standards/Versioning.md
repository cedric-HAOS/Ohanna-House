# Gestion des versions

## Objectif

Ce document décrit la politique de versionnement du projet Ohanna-House.

Chaque version est identifiée par :

- un numéro de version ;
- un nom de code.

Les noms de code sont issus de l'univers Naruto.

---

# Politique de versionnement

## Versions majeures

Les versions majeures représentent une évolution importante de la documentation ou de l'infrastructure.

Elles utilisent le nom d'un personnage emblématique.

Exemples :

| Version | Nom |
|---------|------|
| 1.0 | Naruto |
| 2.0 | Hashirama |
| 3.0 | Minato |

---

## Versions mineures

Les versions mineures correspondent aux évolutions importantes de l'infrastructure.

Le personnage choisi représente symboliquement la nature de cette évolution.

# Règles

- Les noms de code sont définitifs.
- Les versions ne sont jamais renumérotées.
- Chaque version est documentée dans le CHANGELOG.
